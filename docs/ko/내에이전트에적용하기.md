# 드리프트리스를 내 에이전트에 적용하기

이 문서는 드리프트리스를 설치하는 두 가지 방법을 보여 줍니다. 자신에게 맞는 쪽을
고르세요. 두 방법은 정확히 같은 일을 하고, 둘 다 **무언가를 설치하기 전에 반드시
먼저 물어봅니다**(MCP 서버, 플러그인, 추가 도구) — 기본 답은 항상 **아니오**입니다.

코드는 한 줄도 쓸 필요가 없습니다.

---

## 마법 같은 길: 에이전트에게 한 문장 말하기

이 폴더에서 Claude Code나 Codex가 이미 돌고 있다면, 직접 무언가를 실행할 필요가
없습니다. 그냥 에이전트에게 평이한 말로 이렇게 말하세요.

> **"이 저장소를 나한테 적용해줘."**

이렇게 말해도 똑같이 동작합니다.

- "드리프트리스 격리 프로필로 나를 설정해줘."
- "이 레포의 격리 홈을 너한테 설치해줘. 추가 항목은 설치 전에 물어보고."
- "Apply this repo to me." (영어)
- "Set me up with the Driftless isolated profile." (영어)

에이전트가 이 문서를 읽고, 자신이 Claude인지 Codex인지 판단해서 아래 설치
스크립트를 대신 실행합니다. MCP 서버, 플러그인, 의존성을 설치하기 전에는 **반드시
멈추고 물어봅니다.** 대답하지 않으면 답은 "아니오"이고, 아무것도 추가로 설치되지
않습니다.

### 에이전트가 대신 해 주는 일

1. 이 저장소의 루트를 찾습니다(어느 하위 폴더에 있어도 상관없습니다).
2. `./.runtime/` 아래에 에이전트용 **격리 홈**을 만듭니다 — 이 저장소 **안에** 있는
   비공개 설정 폴더입니다.
3. 에이전트가 그 격리 홈을 바라보게 해서, 당신 컴퓨터의 전역 설정을 절대 건드리지
   않게 합니다.
   - Claude는 호스트 전역 `~/.claude` 대신 `CLAUDE_CONFIG_DIR`로 격리 홈을 봅니다.
   - Codex는 호스트 전역 `~/.codex` 대신 `CODEX_HOME`으로 격리 홈을 봅니다.
4. 어떤 CLI가 설치돼 있는지, 다음에 무엇을 하면 되는지 알려 줍니다.

호스트 전역 `~/.claude`와 `~/.codex`는 **절대 읽거나 바꾸지 않습니다.** 모든 것은
이 저장소 안 `./.runtime/`에만 남고, 이 폴더는 git에서 무시되어 절대 커밋되지
않습니다.

## 직접 하는 길: 명령어 한 줄

직접 하고 싶나요? 이 폴더에서 터미널을 열고 명령어 한 줄만 실행하세요.

### macOS / Linux (또는 아무 POSIX 셸)

```sh
# 먼저 계획만 봅니다(아무것도 바꾸지 않음):
sh ./install.sh --dry-run

# 두 프로필 모두 설정:
sh ./install.sh --both

# 또는 하나만:
sh ./install.sh --claude
sh ./install.sh --codex

# 대화식(어떤 도구인지 물어봄):
sh ./install.sh
```

### Windows (PowerShell)

```powershell
# 먼저 계획만 봅니다(아무것도 바꾸지 않음):
pwsh.exe -ExecutionPolicy Bypass -File install.ps1 -DryRun

# 두 프로필 모두 설정:
pwsh.exe -ExecutionPolicy Bypass -File install.ps1 -Tool both

# 또는 하나만:
pwsh.exe -ExecutionPolicy Bypass -File install.ps1 -Tool claude
pwsh.exe -ExecutionPolicy Bypass -File install.ps1 -Tool codex

# 대화식(어떤 도구인지 물어봄):
pwsh.exe -ExecutionPolicy Bypass -File install.ps1
```

이게 설치의 전부입니다. 아무 때나 다시 실행해도 됩니다 — **멱등(idempotent)**
하거든요. 이미 만들어 둔 격리 홈을 재사용하고 빠진 부분만 채웁니다.

## MCP / 플러그인 / 의존성을 설치하기 전에 반드시 물어봅니다

이 부분이 드리프트리스를 안전하고 가볍게 유지하는 핵심입니다. 격리 홈을 설정한
뒤, 설치 프로그램은 질문 세 개를 하나씩 출력합니다.

```
Install optional MCP server(s) for richer tool access? [y/N]:
Install optional plugins into the isolated home?       [y/N]:
Install optional dependencies (extra command-line tools)? [y/N]:
```

- 이 세 질문의 기본값은 모두 **아니오**입니다(대문자 `N`).
- 그냥 Enter만 누르거나, `--dry-run`으로 실행 중이거나, 대화형 터미널이 아니면
  답은 **아니오**가 되고 **아무것도 추가로 설치되지 않습니다.**
- 드리프트리스는 이런 추가 항목 없이도 온전히 동작합니다. 제안만 할 뿐, 멋대로
  가정하지 않습니다.

그래서 MCP 서버나 플러그인, 시스템 패키지가 당신 컴퓨터에 불쑥 나타나는 일은
없습니다. 매번 당신이 결정합니다.

## 실행한 다음

### 3단계 — 격리 홈을 바라보게 해서 에이전트 시작하기

이 저장소 폴더에서 실행하세요. 환경 변수가 전부입니다. 이게 CLI를 저장소 안의
격리 홈으로 향하게 해서, 당신의 호스트 전역 설정은 건드리지 않게 합니다. 설치
프로그램이 당신이 고른 도구에 맞춰 정확히 아래 줄을 출력해 줍니다.

macOS / Linux:

```bash
# Claude
CLAUDE_CONFIG_DIR="$(pwd)/.runtime/claude-home" claude
# Codex
CODEX_HOME="$(pwd)/.runtime/codex-home" codex
```

Windows (PowerShell):

```powershell
# Claude
$env:CLAUDE_CONFIG_DIR="$PWD\.runtime\claude-home"; claude
# Codex
$env:CODEX_HOME="$PWD\.runtime\codex-home"; codex
```

이게 실행 단계입니다 — 따로 설치할 런처 스크립트는 없습니다. 그 환경 변수
자체가 격리입니다. 그다음:

- 당신의 격리 홈은 `./.runtime/` 아래에 있습니다(예: `./.runtime/claude-home`,
  `./.runtime/codex-home`). 그 폴더는 git에서 무시되어, 절대 커밋되거나 공유되지
  않습니다.
- 저장소가 금지 경로를 건드리거나 비밀을 흘리지 않는지 확인하려면 containment
  게이트를 돌리세요.

  ```powershell
  pwsh.exe -ExecutionPolicy Bypass -File scripts/Test-Containment.ps1
  ```

- 함께 들어 있는 스크립트가 모두 Windows에 안전한지(ASCII, BOM 없음) 확인하려면:

  ```powershell
  pwsh.exe -ExecutionPolicy Bypass -File scripts/Test-WindowsTextSafety.ps1
  ```

두 게이트 모두 읽기 전용입니다. 네트워크도, 비밀도, 호스트 전역 접근도 없습니다.

## 에이전트용: apply-driftless 스킬

당신이 "이 레포 나한테 적용해줘"라고 말했을 때 에이전트가 따르는 절차는 공용
**apply-driftless** 스킬(`profiles/shared/skills/apply-driftless/SKILL.md`)입니다.
Claude와 Codex 프로필이 동일하게 사용합니다. 도구를 감지하고, 설치 프로그램을
dry-run으로 돌리고, MCP/의존성/플러그인을 설치하기 전에 물어보고(기본은 아니오),
격리 홈이 제대로 만들어졌고 게이트가 통과하는지 확인한 다음, 평이한 말로
보고합니다. 호스트 전역 설정은 절대 건드리지 않습니다.
