# 드리프트리스(Driftless)란?

**드리프트리스는 Claude와 Codex를 위한, 밤새 스스로 좋아지는 자동 유지보수 도구예요.**

자기 전에 프롬프트(지시문) 하나만 붙여넣으세요. 자는 동안 AI 정비공이 여러분
저장소의 할 일 목록을 꼼꼼한 동료처럼 처리합니다. 열린 이슈를 읽고, 혼자 해도
안전한 일을 골라서, 브랜치에서 작업하고, 풀 리퀘스트(변경 제안)를 만들고, 모든
안전 검사를 통과한 것만 합치죠(머지). 아침에 일어나면 합쳐진 풀 리퀘스트가
기다리고 있어요. **코드는 한 줄도 직접 쓸 필요가 없습니다.**

> **자기 전에 프롬프트 하나 붙여넣기. 일어나면 합쳐진 풀 리퀘스트.**

---

## 아주 쉬운 설명

보통의 "AI 코딩" 도구는 *여러분이* 운전하길 기대해요. 프롬프트를 쓰고, 바뀐
코드를 읽고, 단계마다 승인해 줘야 하죠. 개발자라면 괜찮습니다. 하지만
드리프트리스는 개발자가 **아닌** 사람, 그리고 개발자가 되고 싶지도 않은 사람을
위한 도구예요. *무엇*을 해야 하는지는 알지만 *어떻게* 컴퓨터에게 시키는지는 모르는
관리자를 위한 거죠.

드리프트리스로 여러분이 하는 일은 딱 세 가지입니다.

1. **목표를 한 번 말하기.** 평범한 말로요. "프로젝트를 건강하게 유지하고, 쉬운
   일들은 끝내고, 위험한 건 나한테 먼저 물어봐."
2. **잠자기.** AI 정비공이 엄격한 안전 울타리 안에서 혼자 일합니다
   ([안전장치](./안전장치.md) 참고).
3. **짧은 아침 보고서 읽기.** 여러분의 언어로, 네 가지 분명한 라벨 중 하나로
   시작해요. 완료 / 당신의 결정 필요 / 막힘 / 진행 중. 명령어나 링크 같은 증거는
   쉬운 요약 *뒤에* 붙고, 요약을 대신하지 않습니다.

이 정비공은 자율적으로 움직일 수 있지만, 오직 **울타리 안에서만** 그래요. 스스로
고치고, 브랜치를 만들고, 머지할 수 있습니다. 하지만 여러분의 비공개 파일을 건드릴
수 없고, 돈을 쓸 수 없고, 세상에 무언가를 공개할 수 없고, 되돌릴 수 없는 일도 할 수
없어요. 그런 결정은 항상 짧은 질문으로 여러분에게 돌아옵니다.

---

## 왜 이름이 "드리프트리스"일까요

닻이 없는 배는 *떠내려가요(drift)*. 두고 온 자리에서 멀어지죠. 따로따로 고친 같은
문서 두 개도 점점 *서로 달라지다가(drift apart)* 결국 어긋납니다. AI 에이전트도
*떠내려가요*. 내버려 두면 여러분이 준 목표에서 벗어나, 부탁하지도 않은 것을
"개선"하기 시작하죠.

**드리프트리스는 두 가지 "떠내려가지 않음"을 뜻해요.**

1. **두 개의 도구 프로필이 서로 어긋나지 않아요.** 드리프트리스는 Claude Code와
   Codex를 **하나의 진짜 원본**에서 함께 돌립니다. 안전 규칙, 용어, 공통 스킬이
   딱 한 곳에 있고 두 도구가 그걸 읽어요. 한 번만 고치면 두 도구가 동시에 좋아지죠.
   어긋날 두 번째 복사본이 아예 없습니다. (이건 그냥 바라는 게 아니라 거울 일치
   검사(mirror-parity gate)로 강제됩니다.)
2. **에이전트가 여러분의 목표에서 벗어나지 않아요.** 밤샘 작업은 매번 여러분의 진짜
   의도를 다시 읽고, 결정마다 이유를 기록하고, 숨겨졌거나 확인 안 됐거나 위험한
   일이 남아 있는 동안에는 "완료"라고 말하지 않습니다.

그러니까 "드리프트리스"는 *제자리에 머무른다*는 약속이에요. 두 프로필은 서로
나란히 머물고, 에이전트는 여러분 곁에 머뭅니다.

---

## 여러 도구를 쓰는 건 약점이 아니라 강점이에요

드리프트리스는 일부러 Claude **와** Codex를 둘 다 지원해요. 두 도구에 신경을 나눠
쓰는 게 아니라, 각각의 강점을 쓰는 거예요.

- **Claude Code**는 만들고 고치는 쪽을 잘해요. 변경을 작성하고, 코드를 이해하고,
  밤새 자기 보조 작업들을 지휘하죠.
- **Codex**(`goal` 모드)는 검토하고 출시하는 쪽에 잘 맞아요. 풀 리퀘스트를
  점검하고, 출시를 통제하죠.

요즘의 에이전트 생태계는 일부러 특정 도구에 매이지 않게 만들어져 있어요. **MCP**와
**AGENTS.md** 같은 공통 표준이 업계가 함께 만든 거라서요. 그래서 두 도구를 모두 쓰는
프로젝트는 한 회사에 거는 게 아니라 **생태계의 힘을 빌리는** 거예요. 드리프트리스의
단일 원본 거울 구조 덕분에, 두 번째 도구를 지원하는 비용이 거의 들지 않습니다. 공통
부분은 한 번만 작성하니까요.

## 기술적인 층 (조금 더 깊게)

기술을 아는 분을 위해, 아래에 어떤 장치가 들어있는지 적었어요. 위 내용은 다 쉬운
말로 설명했으니, 이 부분은 그 장치들의 이름을 붙여 두는 정도예요.

### 1. 하나의 원본, 두 개의 프로필

```
driftless/
├── profiles/
│   ├── shared/     <- 단 하나의 원본: 설계 계약, 안전 스키마,
│   │                  도구와 무관한 공통 스킬 (두 도구가 함께 읽음)
│   ├── claude/     <- Claude Code 프로필 (공통 + Claude 전용)
│   └── codex/      <- Codex 프로필 (공통 + Codex 전용)
```

완성된 프로필은 **공통 + 도구 전용**이에요. 두 도구에 똑같은 것(금지 경로 규칙,
증거 용어, 공통 스킬)은 `shared/`에 **한 번만** 있고, 각 프로필은 정말 다른 것만
더합니다. 실행기 방식, 핵심 규칙 파일 이름(`CLAUDE.md` vs `AGENTS.md`), 스킬 형식,
모델 기본값 같은 것들요. **거울 일치 검사**가 두 프로필이 어긋나지 않았는지
확인합니다.

### 2. 다섯 축 경사 하강 (5-axis gradient descent)

드리프트리스는 "더 많은 결과물"을 목표로 하지 않아요. 언덕에서 가장 낮은 곳으로
미끄러져 내려가듯, 측정 가능한 다섯 축을 시간이 갈수록 *낮춰* 갑니다.

| 축 | 뜻 | 목표 |
|---|---|---|
| **토큰** | AI가 한 작업에서 읽고 쓰는 양 | 낮게 |
| **관리자 개입** | 여러분에게 물어봐야 하는 횟수 | 낮게 |
| **시간** | 작업에 걸리는 시간 | 낮게 |
| **돈** | 사용 비용 (정액제 구독이라면 달러가 아니라 요청/세션 수) | 낮게 |
| **성능** | 결과가 실제로 맞는지 | 높게 |

매 작업마다 측정값(텔레메트리)을 남겨서, 추세를 추측이 아니라 눈으로 보게 합니다.

### 3. 강제되는 교훈 승격 사다리 (lesson-promotion ladder)

정비공이 교훈을 배우면, 그 교훈은 잊기 쉬운 메모로만 남지 않아요. 무시했을 때
얼마나 크게 망가지는지에 따라 더 강한 자리로 **승격**됩니다.

```
메모  <  필요할 때 부르는 스킬  <  핵심 규칙  <  훅(hook)  <  게이트 스크립트
 (가장 약함: 기억에만 의존)                  (가장 강함: 기계가 강제, 머지를 막음)
```

되돌릴 수 없거나, 보안과 관련되거나, "가짜 완료"를 일으킬 만한 교훈은 끝까지,
**게이트 스크립트**까지 밀려 올라갑니다. 그 교훈을 지키기 전까지는 작업을 기계적으로
실패시키는 검사죠. 시스템은 배울수록 더 깨기 어려워집니다.

### 4. 비공개 파일을 절대 건드리지 않는 격리 가드

정적 격리 가드가 모든 변경을 검사해서, 어떤 파일의 경로 자체가 금지 구역이거나,
금지 경로를 참조하거나, 자격 증명 토큰을 담고 있으면 **실패(FAIL)** 처리합니다.
컴퓨터의 전역 `~/.claude`나 `~/.codex`, `.env` 파일, SSH 키, 브라우저 프로필, 그
밖의 비밀스러운 것은 절대 읽거나 바꾸지 않아요. 자세한 건
[안전장치](./안전장치.md)를 보세요.

## 이 제품은 스스로를 만들었어요

드리프트리스는 시연용 장난감이 아니에요. 이 도구가 자라난 런타임은 바로 이 밤샘
루프를 **자기 자신의** 저장소에 그대로 썼습니다. 이슈가 브랜치가 되고, 브랜치가 풀
리퀘스트가 되고, 풀 리퀘스트가 검토되고 머지됐어요. 자율 정비공이 끌고, 개발자가
아닌 관리자는 울타리가 쳐진 결정만 승인했죠. 그 원본 저장소의 실제 기록은 이래요.

- **머지된 풀 리퀘스트 113건**
- **이슈 94건**
- **커밋 114건**
- 그 개발 런타임 기준 두 프로필에 걸친 **Claude 스킬 39개 + Codex 스킬 34개**
  (이 공개 키트는 starter 스킬 10개를 ship — `find profiles -name SKILL.md | wc -l`로 확인)
- 깨끗한 트리에서 통과하는 정적 격리 가드, 스크립트의 ASCII + BOM 없음을 강제하는
  Windows 텍스트 안전 게이트, 그리고 프로필 거울 일치 게이트
- 관리자 결정과 에이전트 결정을 이유와 함께 기록한, 날짜가 찍힌 결정 등록부

핵심은 이거예요. 이 루프는 자기 자신을 유지보수할 만큼 진짜라는 것.
