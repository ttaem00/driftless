# 안전장치 (Guardrails)

드리프트리스는 **울타리 안에서 자율적**이에요. 이 한마디가 안전 모델 전부입니다.
AI 정비공은 혼자서 진짜 일을 자유롭게 할 수 있어요. 읽고, 브랜치를 만들고, 고치고,
풀 리퀘스트를 만들고, 통과한 것을 머지하죠. 하지만 넘을 수 없는 울타리 안에서만
그래요. 이 울타리는 AI에게 "지켜 달라"고 부탁하는 권고가 아니에요. 중요한 것들은
경계를 넘으면 **작업을 기계적으로 실패시키는 게이트(검사)**로 강제됩니다.

이 문서는 네 가지 주요 안전장치를 쉬운 말로 설명하고, 그 뒤에 각각을 강제하는 실제
검사 이름을 적어 둡니다.

> **울타리 안에서 자율적:** AI는 *행동하도록* 신뢰받지만, *누구에게* 행동해도
> 되는지를 *결정하도록* 신뢰받지는 않습니다. 그건 울타리가 정하고, 되돌릴 수 없는
> 일은 사람이 정합니다.

---

## 1. 격리 가드 — 여러분의 비공개 파일을 절대 안 건드려요

**쉬운 설명.** 정비공은 오직 여러분의 프로젝트 폴더 안에서만 일해요. 컴퓨터의 비공개
구석에는 절대 손을 뻗지 않습니다. 전역 Claude/Codex 설정, 비밀번호, SSH 키, 브라우저
데이터, `.env` 비밀 같은 것들요. 가드가 "완료"라고 부르기 전에 모든 변경을 검사하고,
그 중 하나라도 그런 곳에 손을 뻗으면 가드가 **실패(FAIL)** 처리하고 작업을 멈춥니다.

**무엇이 금지될까요.** 금지 구역은 두 도구 프로필이 함께 쓰는 단 하나의 기계가 읽는
규칙 모음(`forbidden-paths.json`)이에요. 다음을 막습니다.

- `.env` / `.env.*` 환경 비밀 파일
- `.ssh` 디렉터리 (비공개 SSH 자료)
- `secrets/` 디렉터리
- 비공개 키 (`*.pem` / `*.key`)
- 브라우저 프로필과 자격 저장소 (`Login Data`, `Cookies`, `chrome-profile`)
- **전역** 에이전트 홈 `~/.claude`와 `~/.codex` (정비공은 대신 *저장소 안의* 격리된
  홈을 씁니다. 아래 참고)
- 어떤 파일이든 새어 나온 자격 증명 토큰: GitHub (`ghp_`...), Anthropic
  (`sk-ant-`...), OpenAI 형식 (`sk-`...), AWS (`AKIA`...), 그리고 본문 안의
  `PRIVATE KEY` 블록

**세 가지 종류의 적발.** 가드는 각 문제를 이렇게 보고해요.

- `forbidden_path` — 바뀐 파일의 **경로 자체**가 금지 구역인 경우.
- `forbidden_reference` — 파일의 내용이 금지 경로를 읽거나 쓰는 경우.
- `forbidden_secret` — 파일의 내용에 자격 증명 토큰이 들어 있는 경우.

**격리 불변식 (가장 중요한 약속).** 가드는 비밀 파일이나 금지 파일을 **절대 열지
않아요.** 경로 자체가 금지 구역인 파일은 오직 경로만 보고 적발합니다. 내용은 읽지
않아요. 그래서 가드가 보호하려는 바로 그것들을 가드가 스스로 새어 나가게 만드는 일은
생길 수 없습니다.

**정직한 공정함 (왜 여러분의 문서는 잘못 실패하지 않을까요).** 문서와 가드 자신의
기반 파일(`*.md` 파일들, 게이트 스크립트, 스키마 폴더, `.gitignore`)은 경계를
설명하거나 강제하려고 금지 경로 *이름을 적을 수* 있어요. 지금 이 문서도 `~/.claude`를
적고 있죠. 그래서 그런 파일에서는 `path` 규칙의 *참조*는 적발하지 않습니다. 하지만
**경로-자체 검사와 모든 비밀 규칙은 절대 면제되지 않아요.** 그래서 진짜로 새어 나온
자격 증명은 문서 안에 있어도 잡힙니다.

**삭제가 아니라 격리예요.** 정비공은 여러분의 전역 홈이 아니라 **저장소 안의 격리된
홈**에서 돌아갑니다. Claude는 `CLAUDE_CONFIG_DIR`이 저장소 안 폴더를 가리키고,
Codex는 `CODEX_HOME`이 그래요. 컴퓨터의 전역 `~/.claude`와 `~/.codex`는 전혀 건드리지
않은 채 그대로 둡니다. 저장소 안의 `.claude/`나 `.codex/` 작업 폴더는 프로젝트의
일부라서 경로-자체 검사에서는 면제돼요. 하지만 전역 `~/.claude`를 가리키는 *내용
참조*는 여전히 FAIL이라, 격리를 몰래 우회할 수는 없습니다.

## 2. Windows 텍스트 안전 게이트 — Windows에서 깨지는 스크립트는 안 내보내요

**쉬운 설명.** 스크립트에 잘못된 글자 하나가 들어가면 Windows에서 조용히 망가질 수
있어요. 정비공은 그런 걸 내보낼 수 없습니다. 게이트가 모든 스크립트를 검사하고,
Windows가 잘못 읽을 만한 글자를 찾으면 FAIL 처리합니다.

**왜 중요할까요.** PowerShell 7과 `cmd.exe`는 BOM이 없는 UTF-8 파일을
옛날 CP1252 코드페이지로 읽어요. 길게 늘어진 줄표(em dash), 둥근 따옴표, 또는
라틴 문자가 아닌 글자가 하나라도 있으면 바이트가 깨져 파싱이 실패하죠. 게다가 그
실패는 나중에, 다른 사람 컴퓨터에서야 드러납니다. 게이트는 그걸 미리 잡아요. 추적
중인 모든 `.ps1` / `.bat` / `.cmd`에 대해 세 가지를 강제합니다.

1. **ASCII만 허용 + UTF-8 BOM 없음.** 코드나 문자열 안에 늘어진 줄표, 둥근 따옴표,
   라틴 외 글자 금지. 맨 앞 BOM 금지.
2. **PowerShell 7에서 약한 cmdlet 금지.** 제약된 5.1 호스트에 없을 수 있는
   cmdlet(예: `Get-FileHash`)을 실제로 쓰면, 실행 중에야 실패하기 전에 미리
   적발합니다.
3. **슬래시(/) 훅 경로.** 추적 중인 `settings.json`은 훅의 `command` 경로를 슬래시로
   적어야 해요. 역슬래시(\\) 경로는 Windows에서 뭉개져 데스크톱 에이전트 호스트를
   멈추게 할 수 있습니다.

(이 규칙은 *스크립트*에 적용돼요. 이 문서 같은 *문서 파일*은 한국어, 늘어진 줄표,
풍부한 문장 부호를 자유롭게 써도 됩니다. 셸이 파싱하지 않으니까요.)

## 3. 설치 전에, 컴퓨터를 바꾸기 전에 물어봐요

**쉬운 설명.** 정비공은 소프트웨어를 몰래 설치하거나, 여러분의 전역 설정을 바꾸거나,
프로젝트 밖으로 손을 뻗지 않아요. 컴퓨터를 바꾸거나 외부 도구를 끌어오는 일은
스스로 하는 게 아니라 여러분이 승인하는 요청으로 다룹니다.

실제로는 이래요.

- **기본은 저장소 안.** 설정, 실행 상태, 작업 폴더는 프로젝트 안에 머물러요.
  "전역(global)"은 명시적으로 전역 승격을 허락하지 않는 한, *컴퓨터 전역*이 아니라
  *여기 저장소 전체*를 뜻합니다.
- **갑작스러운 동료(peer)는 없어요.** 정비공은 지금 승인된 작업이 허락하지 않는 한,
  작업 경로에 다른 AI 도구, 다리(bridge), 재귀 에이전트를 끌어오지 않아요. (단,
  여러분 저장소의 일을 처리하려고 자기 보조 작업을 지휘하는 건 허용돼요. 그게 바로
  밤샘 루프 자체니까요.)
- **외부 도구는 기본값이 아니라 결정 사항이에요.** 외부 라이브러리를 도입하거나,
  의존성을 설치하거나, 새 연동을 켜는 일은 추측이 아니라 증거와 함께 여러분에게
  올라옵니다.

## 4. 사람만 정하는 결정 — AI의 몫이 절대 안 되는 결정들

**쉬운 설명.** 짧은 목록의 결정들은 *언제나* 여러분 몫이에요. 정비공이 이런 것에
부딪히면, 추측하지 않고 멈춰서 여러분의 언어로 짧게 물어봅니다. 아무리 "당연히 맞아
보여도" 혼자서는 하지 않아요.

**사람만 정하는 게이트:**

- **제품과 우선순위** — 이 프로젝트가 무엇이 되어야 하는지, 무엇이 가장 중요한지.
- **자격 증명** — 로그인, 키, 비밀과 관련된 모든 것.
- **결제와 사용량 한도** — 돈을 쓰거나 유료 한도를 소모하는 일.
- **공개 출시** — 무언가를 세상에 공개하는 일.
- **파괴적이거나 되돌릴 수 없는 행동** — 삭제, 그리고 되돌릴 수 없는 작업.
- **전역 승격** — 컴퓨터의 전역 설정을 건드리는 일.
- **사용자 데이터 이동** — 여러분의 데이터를 새로운 곳으로 옮기는 일.
- **강제 푸시와 히스토리 초기화** — 공유 프로젝트 기록을 다시 쓰는 일.

이 목록 *바깥*의 것들, 즉 일상적인 GitHub, git, 검증, 안전 작업은 정비공이 스스로
처리해요. 그래서 여러분은 정말로 사람이 필요한 것만 질문받게 됩니다.

## "울타리 안에서 자율적"이 일을 실제로 어떻게 끝낼까요

울타리가 쳐진 자율성은, AI가 울타리를 몰래 건너뛰고 "완료"라고 말할 수 없을 때만
안전해요. 드리프트리스는 그 빈틈을 막습니다.

- **가짜 완료는 없어요.** 숨겨졌거나 확인 안 됐거나 위험한 일이 남아 있는 동안에는
  일이 완료가 아니에요. 확인 안 됐거나 일부만 된 결과는 그렇다고 라벨을 붙이고
  후속 작업으로 추적합니다. 무관한 성공 아래 묻지 않아요.
- **해결하거나 올리거나.** 이번 세션에 AI가 고칠 수 있는 막힘이면, 고치고, 게이트를
  다시 돌리고, 그 재시도를 증거로 보고합니다. 막힘이 위의 사람-전용 게이트면, 대신
  여러분에게 물어봐요.
- **주장보다 증거.** 증거를 만든 진짜 실행 없이는 "통과했다"가 없습니다. 문서를
  정적으로 검사한 건 그 문서를 증명할 뿐, 동작을 증명하지는 않아요.

## 게이트를 직접 돌려보세요

핵심이 되는 두 게이트는 작고 읽기 전용인 PowerShell 스크립트예요. 네트워크도, 비밀도,
전역 접근도 없어요. PowerShell 7과 PowerShell 7에서 똑같이 돌아갑니다.

```powershell
# 격리: 작업 트리의 변경분과 추적되지 않은 파일을 검사 (커밋 전 기본값)
pwsh.exe -ExecutionPolicy Bypass -File scripts/Test-Containment.ps1

# 격리: 추적 중인 모든 파일을 검사 (전체 감사)
pwsh.exe -ExecutionPolicy Bypass -File scripts/Test-Containment.ps1 -AllFiles

# Windows 텍스트 안전: 추적 중인 모든 .ps1 / .bat / .cmd
pwsh.exe -ExecutionPolicy Bypass -File scripts/Test-WindowsTextSafety.ps1
```

두 게이트 모두 `-Json`을 붙이면 기계가 읽는 요약을 줍니다. 종료 코드: `0` = 통과,
`1` = 실패, `2` = 막힘(격리 게이트만. 대상이 git 저장소가 아닐 때. 조용한 통과가
아니라 분명히 막힘으로 보고됩니다).

이 게이트들이 *어느 운영체제에서 검증됐는지*는 영어 문서
**[Host Evidence Matrix](../en/host-evidence-matrix.md)**(운영체제별 정직한 현황)를
보세요. 드리프트리스 전체 소개는 **[드리프트리스란?](./드리프트리스란.md)**에 있어요.
