# 외부 도구 안전하게 들이기

에이전트 생태계는 빠르게 움직이고, 매주 "꼭 써야 한다"는 새 레포·MCP 서버·스킬
묶음이 나옵니다. 드리프트리스는 그런 걸 **반사적으로 갖다 붙이는 걸 막도록** 만들어
졌습니다. 이 문서는 바깥 아이디어를, 드리프트리스를 안전하고 가볍게 만드는 성질을
깨뜨리지 않으면서 들이는 짧은 규칙입니다.

코드를 쓸 필요는 없습니다. 이건 에이전트가 당신을 대신해 따르는 규칙이고, 직접
판단하고 싶다면 한 화면 안에서 읽을 수 있는 규칙입니다.

> **한 문장:** 아이디어를 **가장 작은 형태**로만 빌린다. 프레임워크를 통째로 복사하지
> 않고, 바깥 도구가 울타리 밖으로 손을 뻗지 못하게 한다.

---

## 비개발자에게 왜 중요한가

"이 멋진 레포 적용해줘"라고 말할 때, 위험한 부분은 눈에 안 보입니다. 어떤 도구는
조용히 당신의 전역 설정, 유료 API 키, 돌릴 데이터베이스, 다른 AI를 띄울 권한을
원합니다. 그중 하나라도 들어오면 작고 공짜고 사적인 도구가 무겁고 돈 들고 새는 도구로
바뀝니다. 아래 체크리스트는 그런 일이 생기기 전에 에이전트가 **멈추고 당신에게
보여주게** 합니다 — 그리고 "이 추가 항목 설치할까요?"의 기본 답은 항상 **아니오**입니다.

## 드리프트리스 원칙 (짧게, 알려진 관행에 근거)

널리 인용되는 에이전트 설계 원칙(예: *12-factor-agents* 목록 — 프롬프트를 직접
소유, 컨텍스트를 직접 소유, 작고 집중된 조각, 사람의 개입)을 드리프트리스의 두 가지
현실에 맞춰 변형한 것입니다: **비개발자 주인**, 그리고 **거울처럼 짝지어진 두 프로필
(Claude + Codex)**.

1. **가장 작은 형태가 이긴다.** 아이디어 하나를 문서 한 줄·체크리스트 한 개·작은
   스크립트 한 개로 들인다. 새 인프라가 필요하면 첫날에는 보통 너무 크다.
2. **설치는 먼저 묻고 기본은 아니오.** MCP 서버·플러그인·의존성·패키지는 조용히
   설치되지 않는다. 에이전트가 무엇을·왜 설명한 뒤 당신의 "예"를 기다린다.
3. **울타리는 선택이 아니다.** 들이는 어떤 것도 호스트 전역 설정(`~/.claude`,
   `~/.codex`)이나 비밀(`.env`·`.ssh`·키·브라우저 데이터), 이 프로젝트 격리 홈
   바깥의 무엇도 읽거나 쓸 수 없다.
4. **돈도 재귀도 기본 금지.** 유료 API가 필요하거나, 클라우드로 캡처하거나, 다른
   에이전트를 띄우는 도구는 당신이 명시적으로 결정하지 않는 한 거부한다.
5. **한 번 고치면 양쪽 프로필.** 들인 아이디어가 공용이면 공유 계층에 두어 Claude와
   Codex가 한 곳에서 같이 받게 한다 — 한 프로필에만 복사해 두고 다른 쪽을 잊지 않는다.
6. **정직한 라벨.** 레포의 라이선스·활동성·동작을 실제로 확인하지 않았으면
   `UNVERIFIED`라고 적는다. 안전하다고 추측하지 않는다.

## 에이전트가 레포를 점검하는 법 (가벼운 체크리스트)

공유 스킬 **`adopt-external-tool`** 이 적용 전에 한 화면짜리 점검을 돌립니다. 일부러
설치해야 하는 스캐너가 *아닙니다* — 에이전트가 직접 훑는 체크리스트이고, 끝에 하나의
판정으로 닫습니다:

- 라이선스 공개-안전? • 새 인프라/유료/네트워크? • 호스트 전역·비밀 경로를 만지나? •
  인라인 비밀? • 피어/재귀 AI를 띄우나? • 이 프로젝트에 진짜 ROI? • **가장 작은**
  형태는?

…그다음 하나로: **ADOPT_SMALL**, **PILOT_ONLY**, **WATCH_LATER**(재평가 트리거
포함), **REJECT**(사유 포함), **UNVERIFIED**(못 본 것 포함). 판정은
`docs/external-repo-review.md`에 한 항목으로 추가되어, 같은 레포를 처음부터 다시
따지지 않게 합니다.

에이전트가 외부 스킬·레포·플러그인형 묶음·MCP 설정을 "도입해도 됨"으로 보기 전에는
공개용 사전 도입 게이트도 돌립니다:

```powershell
pwsh.exe -ExecutionPolicy Bypass -File scripts/Test-ExternalAdoptionSafetyGate.ps1 -CandidatePath path\to\candidate
```

이 게이트는 정적 점검에서 임의 실행, 다운로드 후 실행, 호스트 전역·비밀 참조,
자격증명/클라우드/결제/MCP 표면, 데몬 시작, 파일럿 closeout 결정 누락이 보이면 직접
도입을 막습니다. 아무것도 설치하지 않고 악성코드 탐지기도 아닙니다. "도입 전에
멈추는" 확인입니다.

## 증거는 어디에 남나

에이전트가 무언가를 실제로 들였다면, 게이트를 돌려 울타리가 버텼음을 증명합니다 —
컨테인먼트 가드(`Test-Containment.ps1`), 미러 패리티 게이트
(`Test-ProfileMirrorParity.ps1`), 그리고 윈도우 스크립트를 건드렸다면 텍스트 안전
게이트. 못 돌렸으면 "완료"가 아니라 `UNVERIFIED`로 보고합니다.

> 통째로 들이지 않은 선례: *12-factor-agents* 원칙은 여기서 근거로 인용만 합니다.
> 드리프트리스는 그 문서를 복사해 두지 않고, 핫 컨텍스트를 작게 유지하려고 자기만의
> 짧은 규칙을 둡니다.
