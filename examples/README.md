# examples/

Illustrative walkthroughs of what a Driftless run looks like, written for a
non-developer to read. They are **templates, not fabricated live data**: every
issue number, pull-request number, repo name, and count in them is clearly
labeled **EXAMPLE** and was hand-written to show the *shape* of a run — not
captured from a real session on this repository. For what is actually verified
(and on which operating system), see the
[host evidence matrix](../docs/en/host-evidence-matrix.md); for the real history
as it fills in, watch this repo's
[pull requests](https://github.com/mizan0515/driftless/pulls).

## Contents

- **[sample-overnight-run.md](./sample-overnight-run.md)** — one overnight run on
  a small example project: the single parent prompt you paste before bed, then the
  plain-language morning report (done / needs-decision / in-progress / blocked)
  with example merged-PR lines and one risk decision the agent **refused** and
  escalated to you instead of guessing.
