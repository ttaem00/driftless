# 관리자 감사 관점

다음에서 관련 항목을 최소 12개 검사한다.

1. 실제 사용자 기본 경로
2. 요청값과 실제 선택값의 provenance
3. 실패와 fallback의 가시성
4. 중단 작업의 복구와 멱등성
5. 데이터 손실·중복 처리
6. 인증·권한 경계
7. 비용 지불 주체와 사용량 귀속
8. 한도·재시도·백오프
9. 로그의 정확성·민감정보
10. 설정 가능성·하드코딩
11. 롤백과 이전 버전 호환
12. 설치·시작·업데이트 경로
13. 코드맵 최신성·코드 일치
14. 테스트가 실제 동작을 증명하는지
15. 브랜치·PR·메인 반영 여부
16. 생성물·캐시·임시파일 누적
17. 접근성·비개발자 이해 가능성
18. 감시·알림의 소유자와 종료 조건
19. 문서와 UI의 불일치
20. 공개/비공개 정보 분리

모바일처럼 관리자가 명시적으로 제외한 표면은 검사 수를 채우기 위해 다시 포함하지 않는다.
